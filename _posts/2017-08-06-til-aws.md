ec2 (Elastic Cloud Computing)
일반적으로 생각하는 서버 (Linux, Windows)
마이크로 인스턴스는 프리티어에서 사용가능 (월 750시간))
가상 서버이기 때문에 터미널 또는 원격으로 접속해야 함
- Start : 인스턴스 시작. 시작하는 순간 요금이 과금되며 1분을 사용하더라도 1시간 요금으로 책정
- Stop : 인스턴스 정지. 요금이 과금되지 않음
- Terminate : 인스턴스 삭제.
- Reboot : 재부팅
- Root장치 : 운영체제가 설치되는 스토리지. EBS와 인스턴스 스토리지로 나눔
- Kernel ID : 인스턴스가 사용하는 리눅스 커널. 리눅스 반가상화는 외부에서 커널을 지정해야 함.
EC2의 SLA(Service Level Agreement) - 월 99.95%. 즉, 한달에 약 0.36시간 장애가 발생할 수도 있음

EC2 인스턴스 유형
- 범용 : M1, M3로 시작하는 인스턴스. 모든 것이 평균적인 사양으로 제공
- 컴퓨팅 최적화 : C로 시작하는 인스턴스. 다른 것에 비해 CPU비율이 높음
- GPU 인스턴스 : G로 시작하는 인스턴스. NVIDIA GPU 장착. CUDA, OpenCL등 실행할 때 사용
- 메모리 최적화 : M2, CR1으로 시작하는 인스턴스. 다른 것에 비해 메모리 용량이 큼
- 스토리지 최적화 : H, I로 시작하는 인스턴스. 다른 것에 비해 스토리지 용량이 커 초고속 I/O 제공
- 마이크로 인스턴스 : 가격이 가장 싼 인스턴스. 다른 것에 비해 상대적으로 낮은 CPU성능과 낮은 메모리를 제공한다. (프리티어에서 무료사용 가능)

EC2 인스턴스 구매 옵션
- 온디맨드 : 필요할 때 바로 생성해서 사용. 과금은 1시간 단위. 3가지 과금 방식 중 가장 비쌈
  * 공유 인스턴스 : 하나의 물리서버에 여러 개의 EC2 인스턴스 실행. 인스턴스가 자원을 많이 소모한다면 다른 인스턴스 성능에 영향을 줌
  * 전용 인스턴스 : 하나의 물리서버에 하나의 EC2 인스턴스 실행. 다른 인스턴스 성능에 영향을 주지 않는 대신 공유 인스턴스보다 비쌈
- 스팟 : 경매 방식. 가격을 가장 높게 입찰한 사람에게 인스턴스 할당.
(위 옵션은 클러스터링 분석작업, 이미지 변환, 렌더링, 웹 크롤링 등 불시에 중단되어도 상관없는 업무에 적합)
- 예약 : 일정 예약금을 선불로 내면 1년 or 3년동안 예약 가능하며 시간당 요금이 대폭 할인. 온디맨드와 마찬가지로 공유와 전용으로 나뉨
  * Light 사용률 예약 인스턴스 : 가장 저렴. 사용 시간이 많지 않을 때 유용. 1년 중 몇달만 사용하는 개발,테스트와 단기 프로젝트에 적합
  * Medium 사용률 예약 인스턴스 : Light보다 선결제 금액은 비싸지만 시간당 요금이 저렴. 이 유형은 거의 항상 실행하지만, 사용량에 변화가 있을 때 유용
  * Heavy 사용률 예약 인스턴스 : Medium보다 선결제 금액은 비싸지만 시간당 요금이 가장 저렴. 24시간 항상 풀가동 제품에 유용
