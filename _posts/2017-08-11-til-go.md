Go 특징
- 정적 타입, 강 타입
- 컴파일 언어
- 병행성(Concurrency)
- 멀티코어 지원
- 모듈화 / 패키지 시스템
- 빠른 컴파일 속도

정적타입 : 컴파일시 타입 결정 (C, C++, Java, C#)
동적타입 : 런타임시 타입 결정 (Python, Ruby, JavaScript)

약 타입 : 컴파일 시점에 암시적 형 변환 (C)
강 타입 : 값 자체가 타입. 자료형이 다르면 에러

컴파일 언어 : 기계어 형태의 실행 파일을 만들어 낸 후 실행. (속도가 빠르다)
스크립트 언어 : 인터프리터가 소스를 해석하며 실행. (컴파일방식 보다 느림)

go는 c나 c++처럼 완전한 실행파일을 만듦

go는 실행파일에 가비지컬렉터 내장

병행성(Concurrency) : 논리적인 동시 처리. 시분할과 같은 동시처리처럼 보이지만 내부적으로 시간을 쪼개 실행하는 형태도 논리적으로는 병렬처리와 같음.
병렬성(Parallelism) : 물리적인 동시 처리. 작업을 CPU코어에 나눠서 동시 처리.

go는 `go`키워드를 통해 함수 여러 개 동시 실행 가능 - 이를 고루틴(goroutine)이라 함.

고루틴/스레드 차이
스레드는 커널에서 제공하는 리소스. 많이 생성할수록 부담이 늘어남
스레드 위에 고루틴이 올라가는 구조. 스레드가 고루틴을 처리한다.
고루틴끼리 통신 가능

모듈 및 패키지
`go get`/`go install`

go는 c/c++과 달리 헤더파일이 없어 컴파일 속도가 빠름
소스 코드를 패키지화하여 변경된 부분만 컴파일

go는 크고 복잡한 어플리케이션 개발에 적합
자원관리보다 로직에 집중할 수 있는 구조
즉, 메모리 관리를 덜 해도 되고, 규모가 크고 복잡하며 유지보수가 빈번한 곳에서 사용하는 것이 적합

go패키지구조
bin,pkg,src로 구성
bin : 패키지를 컴파일하여 실행파일이 생성되는 곳
pkg : 패키지를 컴파일하여 라이브러리 파일이 생성되는 곳 (<운영체제>_<아키텍처>형태로 생성 `linux_amd64`)
src : 내가 작성한 소스파일과 인터엣에서 자동으로 받아온 소스파일이 있는 곳

GOPATH설정
GOPATH를 환경변수로 설정하면 컴파일, 라이브러리등 해당 디렉토리를 기준으로 함.
`go env`로 설정된 GOPATH확인 가능
GOPATH는 일종의 workspace라고 보면 됨. 프로젝트가 여러개이면 그때그때 바꿔주어야 함

빌드는 `go build`로 함

go는 작성 스타일을 강제하는 문법이 있음
함수, 조건문, 반복문을 사용할 때 쓰는 `{}`는 반드시 같은 줄에서 시작해야 함

```
func main() {
    i := 10

    if i >= 5 {
        fmt.Println("5 이상")
    }

    for i := 0; i < 5; i++ {
        fmt.Println(i)
    }
}
```

`gofmt` 문법 스타일 자동 맞춤 기능
한줄에 여러개의 구문이 있을 때 `;`로 구문을 구분함. 라인이 변경되면 세미콜론을 사용하지 않아도 됨

go의 들여쓰기는 탭 사용

* 콘솔에서 gofmt사용방법
`gofmt -w hello.g` -w는 정렬한 내용을 저장함

go 변수 선은은 특이하게 타입이 뒤로 온다.

```
var i int
var age int = 10
var name string = "Kim"
```

자료형을 생략하면 대입하는 값의 자료형으로 결정

```
var age = 10
var name = "Kim"
```

짧은 선언 사용하기
`age := 10`

변수 여러개 선언
`x, y, z = 10, 20, 5`

var와 ()를 사용하면 변수 여러개 선언 가능

```
var (
    x, y      int = 30, 50      // x와 y는 int형으로 결정
    age, name     = 10, "Maria" // age는 int, name은 string으로 결정
)
```

사용하지 않는 변수 오류 발생
사용하지 않으려면 변수명을 `_`로 사용
패키지도 사용하지 않을 때는 `_`을 붙임
