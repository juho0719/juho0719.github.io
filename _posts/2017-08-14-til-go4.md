함수
자바와는 다르게 리턴 변수명을 미리 지정하면 리턴할 때 변수명 안적어도 됨
```
func sum(a int, b int) (c int) {
    c = a + b 
    return    // 리턴값 변수를 사용할 때는 return 뒤에 변수x
}
```

리턴값 여러개 사용가능
```
func sum(a int, b int) (c int, d int) {
c = a + b 
d = a - b
return
}
```

가변인자
```
func sum(n ...int) int { // int형 가변인자
    total := 0
    for _, value := range n { 
        total += value   
    }

    return total
}
```

...을 사용하면 슬라이스의 각 요소들을 전달함
```
n := []int{1, 2, 3, 4, 5}
r := sum(n...)  // sum인자값에 슬라이스 각 요소들을 넘겨줌
```

클로저
함수 안에서 함수를 정의할 수 있고, 바깥쪽 함수에 선언된 변수에도 접근할 수 있는 함수
(보통 구성할 때 익명 함수를 사용)
```
func calc() func(x int) int {
    a, b := 1, 2                // 지역 변수는 함수가 끝나면 소멸되지만
    return func(x int) int {    // 클로저
        return a*x + b          // 클로저이므로 함수를 호출 할 때마다 변수 a와 b의 값을 사용할 수 있음
    }
    // 익명 함수를 리턴
}
```

panic : 사용자가 직접 에러 발생
recover : 패닉 발생시 프로그램 종료가 아닌 예외처리 가능 (try-catch의 catch와 같음)
```
func f() {
    defer func() {         // recover 함수는 지연 호출로 사용해야 함. 아니면 패닉 후 바로 종료됨
        s := recover()     // panic 함수에서 설정한 에러 메시지를 받아옴
        fmt.Println(s)
    }()

    panic("Error !!!") // panic 함수로 에러 메시지 설정, 패닉 발생
}
```

포인터
c언어와 달리 변수 선언시 *를 자료형에 붙임
`var a *int`

최초 선언시 `nil`로 초기화 됨 (go에선 null이 nil로 사용. c에서는 null은 정수 0이 정의 되어 있지만 go에서는 nil 0이 아니다.)

빈 포인터는 바로 사용할 수 없고 new로 초기화 해야 함.(go는 가비지컬렉터가 있어 따로 해제 하지 않아도 됨)
`var a *int = new(int)`

변수를 사용할 때 *는 역참조다.(포인터 변수에 값을 넣기 위해서는 역참조하여 대입)
`*a = 1     // 포인터변수가 가르키는 메모리에 1을 대입`

다른 일반 변수를 포인터변수에 대입할 때는 &를 사용. 이때는 call by reference
```
var a int = 1
var b *int = &a
```

go는 메모리 주소를 직접 대입하거나 포인터 연산을 허용하지 않음

구조체를 함수의 인자값으로 사용할 때는 주소값을 넘겨주어야 반영된다. (구조체를 넘기면 값 복사가 이루어져 넘긴 구조체의 값이 변화하지 않는다.)

구조체에 메서드 연결
`func`와 함수명 사이에 리시버 정의 영역이 추가됨. 구조체 변수를 통해 해당 메서드 접근 가능
메서드안에서 리시버 변수를 통해 해당 구조체 값에 접근할 수 있음.
```
func (rect *Rectangle) area() int {     //rect : 리시버 변수명
    return rect.width * rect.height     //리시버 변수를 통해 구조체 값 접근 가능
}
```

구조체 임베딩
go는 클래스를 사용하지 않아 상속 또한 없음. 대신 임베딩(Embedding)을 사용하면 상속과 같은 효과를 낼 수 있음
```
type Person struct { // 사람 구조체 정의
    name string
    age  int
}

func (p *Person) greeting() { // 인사(greeting) 함수 작성
    fmt.Println("Hello~")
}

type Student struct {
    p      Person // 학생 구조체 안에 사람 구조체를 필드로 가지고 있음. Has-a 관계
    school string
    grade  int
}

func main() {
    var s Student

    s.p.greeting() // Hello~
}
```
학생 구조체 안에 Person구조체를 가지고 있는 Has-a관계 성립 (임베딩아님)

```
type Person struct {
    name string
    age  int
}

func (p *Person) greeting() {
    fmt.Println("Hello~")
}

type Student struct {
    Person // 필드명 없이 타입만 선언하면 포함(Is-a) 관계가 됨
    school string
    grade  int
}

func main() {
    var s Student

    s.Person.greeting() // Hello~
    s.greeting()        // Hello~
}
```
Person구조체를 선언할 때 필드명을 정의 하지 않는 것을 임베딩이라 하며, Is-a관계가 성립.
Student에서 바로 greeting()을 호출 할 수 있음
필드명을 정의 하지 않는 필드를 익명 필드라 함.

만약 Student에 greeting()이라고 같은 이름으로 메소드를 정의하면 오버라이딩됨.
해당 구조체의 메소드를 호출하려면 해당 타입을 지정해야 함.
```
s.Person.greeting() // Hello~
s.greeting()        // Hello Students~
```

인터페이스 선언
```
type 인터페이스명 interface {
    메서드1() 리턴값_자료형
    메서드2()                  // 리턴값이 없을 때
}
```

각각의 타입이나 구조체에 연결된 메소드를 인터페이스에 선언하여 사용하면 해당 인터페이스 변수에 해당 값을 대입하면 실제 연결된 메소드에 값이 대입됨.
```
type MyInt int // int 형을 MyInt로 정의

func (i MyInt) Print() { // MyInt에 Print 메서드를 연결
    fmt.Println(i)
}

type Rectangle struct { // 사각형 구조체 정의
    width, height int
}

func (r Rectangle) Print() { // Rectangle에 Print 메서드를 연결
    fmt.Println(r.width, r.height)
}

type Printer interface { // Print 메서드를 가지는 인터페이스 정의
    Print()
}

func main() {
    var i MyInt = 5
    r := Rectangle{10, 20}

    var p Printer // 인터페이스 선언

    p = i     // i를 인터페이스 p에 대입
    p.Print() // 5: 인터페이스 p를 통하여 MyInt의 Print 메서드 호출

    p = r     // r을 인터페이스 p에 대입
    p.Print() // 10 20: 인터페이스 p를 통하여 Rectangle의 Print 메서드 호출
}
```

배열이나 슬라이스 형태로도 인터페이스를 초기화 할 수 있음
```
var i MyInt = 5
r := Rectangle{10, 20}

pArr := []Printer{i, r} // 슬라이스 형태로 인터페이스 초기화
for index, _ := range pArr {
    pArr[index].Print() // 슬라이스를 순회하면서 Print 메서드 호출
}

for _, value := range pArr {
    value.Print() // 슬라이스를 순회하면서 Print 메서드 호출
}
```

이렇게 각 값이나 인스턴스의 실제 타입은 상관하지 않고 구현된 메소드만으로 판단하는 방식을 덕 타이핑(Duck typing)이라 함

해당 타입이 특정 인터페이스를 구현하는지 검사하려면 `interface{}(인스턴스명).(인터페이스)` 로 함
```
var whiteTiger Animal

if v, ok := interface{}(whiteTiger).(Animal); ok {
    fmt.Println(v, ok)
}
```
첫번째 리턴 값은 검사했던 인스턴스이고, 두번째 리턴값은 인스턴스가 해당 인터페이스를 구현하고 있는 지 여부임.

빈 인터페이스 사용하기
```
func a(arg interface{}) {   // 모든 타입 저장 가능

}

type Any interface{}
func b(arg Any) {       // Any로 표현

}
```

모든 타입을 다 받을 수 있으므로, arg.(타입)형태로 처리 해야 함.
```
func formatString(arg interface{}) string {
    switch arg.(type) {
    case int:                   // arg가 int형이라면
        i := arg.(int)          // int형으로 값을 가져옴
        return strconv.Itoa(i)  // strconv.Itoa 함수를 사용하여 i를 문자열로 변환
    case float32:               // arg가 float32형이라면
        f := arg.(float32)      // float32형으로 값을 가져옴
        return strconv.FormatFloat(float64(f), 'f', -1, 32)
    case Person:                // arg의 타입이 Person이라면
        p := arg.(Person)       // Person 타입으로 값을 가져옴
        return p.name + " " + strconv.Itoa(p.age)
    case *Person:               // arg의 타입이 Person 포인터라면
        p := arg.(*Person)      // *Person 타입으로 값을 가져옴
        return p.name + " " + strconv.Itoa(p.age)
    default:
        return "Error"
    }
}
```

고루틴
함수를 동시에 실행시키는 기능. 자바의 스레드 생성보다 문법이 간단하고 리소스를 더 적게 사용
```
func hello() {
    fmt.Println("Hello, world!")
}

func main() {
    go hello()      // 함수를 고루틴으로 실행
    fmt.Scanln()    // main 함수가 종료되지 않도록 대기. 아니면 main이 너무 빨리 끝나 고루틴이 실행되기도 전에 끝남
}
```

멀티코어 활용
```
func main() {
    runtime.GOMAXPROCS(runtime.NumCPU()) // CPU 개수를 구한 뒤 사용할 최대 CPU 개수 설정
    fmt.Println(runtime.GOMAXPROCS(0)) // 설정 값 출력
}
```
runtime.GOMAXPROCS()는 특정 값을 설정해도 됨. 0을 넣으면 설정 값을 바꾸지 않고 현재 값을 리턴

클로저를 고루틴으로 실행할 때 변수 사용에 주의해야 함. 고루틴으로 실행한 클로저는 반복문이 끝난뒤에 실행됨.
반복문에 의해 바뀌는 변수는 반드시 매개변수로 넘겨주어야 함.
```
s := "Hello, world!"

for i := 0; i < 100; i++ {
    go func() {
        fmt.Println(s, i)   // 반복문의 변수를 클로저에서 출력하므로 반복문이 다 끝난 후 고루틴이 실행되면 100만 출력됨
                            // 클로저에서 바로 출력
    }() //매개변수로 넘기지 않음
}
```

채널 사용하기
고루틴끼리 데이터를 주고 받고, 실행 흐름을 제어함
`a := make(chan int)    //int형 채널 생성`

채널에 값을 넣을 때
`a <- i + j`

채널에서 값을 받을 때
`n := <-a`

채널에 값을 넣으면 값을 꺼낼 때까지 대기한다. (동기채널)

채널 버퍼링
채널의 버퍼가 가득차면 값을 꺼내 출력
버퍼를 1개이상 설정하면 비동기 채널이 생성. 비동기 채널은 보내는 쪽에서 버퍼를 다 채우면 실행을 멈추고 받는 쪽에서 버퍼에 값이 없으면 대기.
`make(chan 자료형, 버퍼개수)`

range를 통해 채널에 담긴 값들을 추출 할 수 있음.
채널에 들어올 값의 개수를 알 수 없기 때문에 들어 올 때마다 계속 꺼내기 위해 사용
close()를 통해 채널을 닫으면 range도 종료
두번째 리턴값으로 채널 닫힘여부 확인 가능
`a, ok := <-c   //1, true: 채널은 열려있고 1을 꺼냄`

단방향 채널
보내기용, 받기용 채널을 고정할 수 있음 (<-위치에 따라)
```
func producer(c chan<- int) { // 보내기 전용 채널
    for i := 0; i < 5; i++ {
        c <- i
    }

    c <- 100           // 채널에 값을 보냄

    //fmt.Println(<-c) // 채널에서 값을 꺼내면 컴파일 에러
}

func consumer(c <-chan int) { // 받기 전용 채널
    for i := range c {
        fmt.Println(i)
    }

    fmt.Println(<-c) // 채널에 값을 꺼냄

    // c <- 1        // 채널에 값을 보내면 컴파일 에러
}

func main() {
    c := make(chan int)

    go producer(c)
    go consumer(c)

    fmt.Scanln()
}
```

리턴 값에도 지정 가능
```
func sum(c <-chan int) <-chan int {
    out := make(chan int) // int형 채널 생성
    go func() {
        r := 0
        for i := range c { // range를 사용하여 채널이 닫힐 때까지 값을 꺼냄
            r = r + i  // 꺼낸 값을 모두 더함
        }
        out <- r           // 더한 결과를 채널에 보냄
    }()
    return out // 채널 변수 자체를 리턴
}

```

동기화 객체 사용
- Mutex : 여러 고루틴에서 공유되는 데이터 보호
- RWMutex : 읽기/쓰기 뮤텍스. 읽기와 쓰기를 나누어서 락을 걸 수 있음
- Cond : 조건 변수. 대기하고 있는 객체를 깨움(여러개 가능)
- Once : 특정 함수를 한번만 실행
- Pool : 멀티 고루틴에서 사용하는 풀. 자주 사용하는 객체를 재사용
- WaitGroup : 고루틴이 모두 끝날 때까지 대기
- Atomic : 멀티 고루틴/멀티 코어 환경에서 안전하게 값을 연산

뮤텍스
```
sync.Mutex
func (m * Mutex) Lock() : 락
func (m * Mutex) UnLock() : 언락
```

runtime.Gosched : 다른 고루틴이 CPU를 사용할 수 있도록 양보

```
func main() {
    runtime.GOMAXPROCS(runtime.NumCPU()) // 모든 CPU 사용

    var data = []int{}
    var mutex = new(sync.Mutex)

    go func() {                     // 고루틴에서
        for i := 0; i < 1000; i++ { // 1000번 반복하면서
            mutex.Lock()            // 뮤텍스 잠금, data 슬라이스 보호 시작
            data = append(data, 1)  // data 슬라이스에 1을 추가
            mutex.Unlock()          // 뮤텍스 잠금 해제, data 슬라이스 보호 종료

            runtime.Gosched()       // 다른 고루틴이 CPU를 사용할 수 있도록 양보
        }
    }()

    go func() {                     // 고루틴에서
        for i := 0; i < 1000; i++ { // 1000번 반복하면서
            mutex.Lock()            // 뮤텍스 잠금, data 슬라이스 보호 시작
            data = append(data, 1)  // data 슬라이스에 1을 추가
            mutex.Unlock()          // 뮤텍스 잠금 해제, data 슬라이스 보호 종료

            runtime.Gosched()       // 다른 고루틴이 CPU를 사용할 수 있도록 양보
        }
    }()

    time.Sleep(2 * time.Second) // 2초 대기

    fmt.Println(len(data)) // data 슬라이스의 길이 출력
}
```

읽기/쓰기 뮤텍스
읽기 락 : 읽기 시도중에 값이 바뀌면 안되므로 쓰기 락 상태
쓰기 락 : 쓰기 시도중에 다른 곳에서 이전 값을 읽으면 안되고, 다른 곳에서 값을 바꿔도 안되므로 둘다 락 상태
읽기 락끼리는 막지 않아 여러 코어에서 읽기 가능

조건 변수
```
go func(n int) {                    
    mutex.Lock()                    // 뮤텍스 잠금, cond.Wait() 보호 시작
    c <- true                       // 채널 c에 true를 보냄
    fmt.Println("wait begin : ", n)
    cond.Wait()                     // 조건 변수 대기
    fmt.Println("wait end : ", n)
    mutex.Unlock()                  // 뮤텍스 잠금 해제, cond.Wait() 보호 종료

}(i)
```

1개 깨우기
```
mutex.Lock()                // 뮤텍스 잠금, cond.Signal() 보호 시작
fmt.Println("signal : ", i)
cond.Signal()               // 대기하고 있는 고루틴을 하나 깨움
mutex.Unlock()              // 뮤텍스 잠금 해제, cond.Signal() 보고 종료
```

여러개 깨움
```
mutex.Lock()             // 뮤텍스 잠금, cond.Broadcast() 보호 시작
fmt.Println("broadcast")
cond.Broadcast()         // 대기하고 있는 모든 고루틴을 깨움
mutex.Unlock()           // 뮤텍스 잠금 해제, cond.Signal() 보고 종료
```

wait, signal, broadcast 모두 Lock/Unlock으로 보호 되어야 함.

풀 사용
일종의 캐시.
수명이 짧은 객체는 풀에 적합하지 않음

풀 할당
```
pool := sync.Pool{                            // 풀 할당
    New: func() interface{} {             // Get 함수를 사용했을 때 호출될 함수 정의
        data := new(Data)             // 새 메모리 할당
        data.tag = "new"              // 태그 설정
        data.buffer = make([]int, 10) // 슬라이스 공간 할당
        return data                   // 할당한 메모리(객체) 리턴
    },
}
```

`data := pool.Get().(*Data) // 풀에서 *Data 타입으로 데이터를 가져옴`

풀 보관
`pool.Put(data)`

대기 그룹 사용
고루틴이 끝날 때까지 대기
```
wg := new(sync.WaitGroup) // 대기 그룹 생성

for i := 0; i < 10; i++ {
    wg.Add(1)            // 반복할 때마다 wg.Add 함수로 1씩 추가
    go func(n int) {     // 고루틴 10개 생성
        fmt.Println(n)
        wg.Done()    // 고루틴이 끝났다는 것을 알려줌
    }(i)
}

wg.Wait() // 모든 고루틴이 끝날 때까지 기다림
fmt.Println("the end")
```

Add,Done,Wait로 이루어짐. Add로 고루틴 생성시 추가해주고, 끝나면 Done을 사용.
Wait을 사용하면 모든 고루틴이 끝날 때까지 대기함
Done은 `defer`와 함께 사용하여 지연 호출로 사용하기도 함

Atomic 사용
멀티 고루틴/멀티 코어 환경에서 변수를 수정할 때 영향받지 않고 안전하게 연산
`atomic.AddInt32(&data, 1) // 원자적 연산으로 1씩 더함`

atomic패키지 연산 종류
- Add 계열 : 변수에 값을 더하고 결과 리턴
- CompareAndSwap 계열 : A와 B를 비교하여 같으면 C대입, A와 B가 같으면 true, 다르면 false
- Load 계열 : 변수에서 값을 가져옴
- Store 계열 : 변수에 값을 저장
- Swap 계열 : 변수에 새 값을 대입하고, 이전 값을 리턴
